# HTML 到 Theme JSON 转化与手绘主题生成计划

## 1. 转化过程分析 (HTML -> JSON)

通过对比 `design-demo.html` 和 `pixel-theme.json`，我们可以总结出以下转化规则：

1. **样式提取 (Style Extraction)**: 将 HTML 元素的内联样式（或 CSS 类样式）提取出来。
2. **属性转换 (Property Conversion)**: 将 CSS 属性名（如 `background-color`）转换为 JSON 的驼峰命名格式（如 `backgroundColor`）。
3. **结构映射 (Structural Mapping)**:

   * **容器映射**: HTML 中的 `<section>` 或 `div` 容器对应 JSON 中的 `wrapper`, `section`, `h1Container` 等对象。

   * **组件映射**: 特定的 UI 模式被抽象为组件配置。例如：

     * `blockquote` 内部的标签 -> `blockquoteBadge`

     * 列表项前的图标 -> `ulMarker` / `olMarker`

     * 代码块头部 -> `preHeader`
4. **值保留**: 复杂的 CSS 值（如 `box-shadow`, `linear-gradient`, `transform`）直接作为字符串保留。

## 2. 手绘风格主题生成计划

基于上述规则，我将把 `hand-drawn-cartoon-style.html` 转换为 `themes/hand-drawn-theme.json`。

### 关键组件映射方案：

* **全局 (Global)**:

  * `.wrapper` -> `wrapper`: 保留 `border-radius: 20px`, `box-shadow`, `border`。

  * **注意**: 原 HTML 中的 `::before` 和 `::after` (涂鸦装饰) 无法直接映射到当前的 JSON 结构中，将被省略或寻找替代方案。

* **标题 (Headings)**:

  * `h1` -> `h1`: 保留旋转角度 `transform: rotate(-1deg)` 和渐变背景。

  * `h2` -> `h2`: 保留便签纸风格和旋转。

  * `h3` -> `h3`: 保留虚线下划线。

  * `.h3-badge` -> `h3Badge`: 对应 TIP 标签。

* **文本 (Text)**:

  * `strong`: 对应荧光笔高亮效果。

  * `a`: 对应波浪下划线 `border-bottom: 2px wavy`。

  * `code`: 对应便利贴风格。

* **引用与列表 (Blockquote & Lists)**:

  * `blockquote`: 对应对话气泡风格。

  * `ulMarker`: 提取其中一种颜色（如粉色）作为默认样式。

  * `olMarker`: 提取手绘数字圆圈样式。

* **代码与表格 (Pre & Table)**:

  * `.pre-container` -> `pre`: 黑板风格。

  * `.table-container` -> `tableContainer`: 手绘网格风格。

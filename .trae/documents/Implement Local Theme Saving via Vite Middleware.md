# 实现本地主题保存及全量预览功能计划

本计划旨在完善主题提取工作流，允许用户在本地保存提取的主题，并在保存前通过全量预览确认样式效果。

## 1. 后端实现：Vite 中间件 (Vite Middleware)

利用 Vite 的插件机制创建一个本地开发服务器中间件，提供文件写入能力。

-   **目标文件**: `vite.config.ts`
-   **功能**:
    -   创建 `saveThemePlugin` 插件。
    -   拦截 `POST /api/save-theme` 请求。
    -   解析请求体中的 `theme` 对象和 `filename`。
    -   验证文件名（必须包含 `.json` 后缀）。
    -   将主题数据格式化为 JSON 字符串并写入项目的 `themes/` 目录。
    -   返回 200 成功或 500 错误状态。

## 2. 前端实现：主题提取器 UI 升级

改造 `components/ThemeExtractorUI.tsx`，从单纯的“提取-下载”变为“提取-预览-保存”。

### 2.1 全量预览数据 (`PREVIEW_MARKDOWN`)
构造一个包含所有标准元素的 Markdown 字符串，确保覆盖：
-   **标题**: H1 (主/副), H2, H3 (带徽章), H4, H5
-   **文本**: 正文, **加粗**, *斜体*, `行内代码`, [链接]
-   **列表**: 无序列表 (ul), 有序列表 (ol)
-   **容器**: 引用块 (blockquote), 代码块 (pre), 表格 (table)
-   **分隔**: 分割线 (hr)
-   **页脚**: Footer 样式

### 2.2 界面交互更新
-   **引入组件**: 导入 `WeChatRenderer`。
-   **布局调整**: 在 `step === 'complete'` 状态下，将界面分为左右两栏（或上下布局，视空间而定）：
    -   **左侧**: 提取结果统计、警告信息。
    -   **右侧**: 预览区域，使用 `WeChatRenderer` 渲染 `PREVIEW_MARKDOWN` 和提取出的 `theme`。
-   **保存功能**:
    -   添加“保存到项目 (SAVE)”按钮。
    -   点击触发弹窗或输入框，让用户输入主题名称（如 `my-theme`）。
    -   调用 `/api/save-theme` 接口。
    -   成功后显示 Toast 或提示信息。

## 3. 验证步骤

1.  **环境准备**: 停止当前服务，重新运行 `npm run dev` 以加载新的 Vite 配置。
2.  **提取测试**: 上传一个包含样式的 HTML 文件。
3.  **预览确认**: 在结果页检查右侧预览区，确认 H1-H5、列表、表格等所有元素样式是否正确应用。
4.  **保存测试**: 点击“保存”，输入测试名称 `test-save-feature`。
5.  **结果验证**: 检查本地项目根目录下的 `themes/` 文件夹，确认 `test-save-feature.json` 是否存在且内容正确。

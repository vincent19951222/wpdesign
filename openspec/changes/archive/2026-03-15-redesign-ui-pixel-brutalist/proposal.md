## Why

当前产品的界面风格仍然是多套视觉语言混用，首页、编辑器、预览区和基础控件之间缺少统一的设计系统，导致品牌识别度和界面一致性不足。现在需要用一套明确的复古像素风设计系统替换现有 UI 基线，让产品在保留现有功能流程的前提下，形成稳定、可复用、可持续扩展的视觉与交互规范。

## What Changes

- 以统一的复古像素风设计系统重构当前前端界面，替换现有混杂的视觉样式与局部控件表现。
- 定义一套共享的颜色、字体、边框、阴影、背景和像素网格规范，作为全站 UI 的唯一视觉基线。
- 新增可复用的基础像素类和控件样式约定，覆盖面板、按钮、输入框等核心交互元素。
- 建立严格的字体分工：中文统一使用 `Fusion Pixel`，英文大标题使用 `Press Start 2P`，英文次级文本与按钮使用 `Silkscreen`。
- 将主要页面和核心组件迁移到新的视觉系统，包括应用外壳、主题选择区、编辑区、预览区及其相关操作控件。
- 保持现有业务能力和内容工作流不变，本次变更聚焦视觉重构与交互表现统一，不引入新的后端能力或发布流程变化。

## Capabilities

### New Capabilities

- `pixel-brutalist-design-system`: 定义复古像素风 UI 的颜色、字体、阴影、边框、背景纹理、像素网格和基础复用样式约束，作为全站统一设计令牌与样式基线。
- `pixel-brutalist-interface-refresh`: 要求关键页面和共享组件采用新的像素风设计系统，统一面板、按钮、输入框和核心内容工作流界面的视觉与交互表现。

### Modified Capabilities

- None.

## Impact

- Affected code: `tailwind.config.js`、全局样式文件、应用入口样式注入点、共享 UI 组件、首页/工作台/预览等核心前端页面与组件。
- Dependencies: 需要引入并管理 `Fusion Pixel`、`Press Start 2P`、`Silkscreen` 等字体资源，以及对应的全局字体应用与像素网格约束策略。
- Systems: 影响当前前端展示层和设计系统基线，不涉及后端接口、Markdown 渲染逻辑或微信草稿同步能力变更。
